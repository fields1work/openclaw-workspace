## 🎛️ OPERATIONAL MODES

### Execution Mode
**Entry Trigger:** Clear task, no blockers, resources available  
**Allowed Actions:** Implement, ship, post, optimize  
**Exit Criteria:** Task complete OR blocker encountered → switch mode

### Research Mode
**Entry Trigger:** New tool, new trend, or "how do we do X?"  
**Allowed Actions:** Read docs, test, document findings, create templates  
**Exit Criteria:** Research complete → switch to Execution  
**Time Box:** Max 2 hours per research sprint

### Debug Mode
**Entry Trigger:** Something is broken  
**Allowed Actions:** Diagnose, document, fix, verify, prevent  
**Exit Criteria:** Root cause identified AND fix verified OR escalated  
**Time Box:** Max 4 hours per incident

### Optimization Mode
**Entry Trigger:** Process working but inefficient OR new learning acquired  
**Allowed Actions:** Streamline, automate, template, eliminate  
**Exit Criteria:** Improvement < 15% → document and move on

### Expansion Mode
**Entry Trigger:** Current system stable, capacity available  
**Allowed Actions:** New platforms, new niches, new tools  
**Exit Criteria:** New platform showing traction OR 30 days with no traction

### Incident Response Mode
**Entry Trigger:** Critical system down (revenue blocked)  
**Allowed Actions:** Stop everything, fix, document, harden  
**Exit Criteria:** System restored, root cause documented, prevention added

**Current Mode:** Execution → incident response → execution  
*(GitHub auth issue, browser extension fix)*
